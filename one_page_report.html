IoT Network Intrusion Detection Using Machine Learning

Cybersecurity — Machine Learning Project Report
Group Members: Marc Boulos  |  Anthony Farhat  |  Jad Allam  |  Karl Nguyen Van Doan

Data Source

We used the CICIoT2023 dataset published by the Canadian Institute for Cybersecurity (University of New Brunswick). It contains labeled network traffic flows captured from real IoT devices under both normal and attack conditions, spanning 9 classes: Benign, DDoS, DoS, Mirai, Spoofing, Recon, Recon-HostDiscovery, Web, and DictionaryBruteForce. The dataset provides 46 numerical flow-level features (packet rates, flag counts, header lengths, etc.).

Approach

We built a multi-class streaming intrusion detection system (IDS) that classifies IoT network traffic flows in real time and monitors for data drift. The pipeline covers data ingestion, preprocessing, model training, evaluation, and deployment as a REST API served inside a Docker container. We applied SMOTE to address severe class imbalance and used Macro-F1 as our primary metric, since accuracy is misleading on imbalanced data.

Summary of Methods

Preprocessing: Stratified 70/15/15 train/val/test split, median imputation for missing values, standard scaling, SMOTE oversampling on the training set only.

Models trained: Logistic Regression (baseline), Random Forest, XGBoost, LightGBM, and a custom PyTorch MLP neural network. Hyperparameters were tuned on the validation set.

Deployment: The best model is served via a FastAPI REST API with endpoints for single and batch prediction, confidence thresholding (fallback to Benign below 70% confidence), and live drift monitoring using KS-test, PSI, and Jensen-Shannon divergence. The full system is containerised with Docker and published on Docker Hub (marcboulos/iot-ids:latest).

LightGBM achieved the best Macro-F1 of 0.830 and ROC-AUC of 0.996, making it the deployed model. Ensemble tree methods (LightGBM, XGBoost, Random Forest) significantly outperformed the linear baseline and the neural network on this structured tabular dataset. The system successfully detects all 8 attack types with a false positive rate below 9%.
